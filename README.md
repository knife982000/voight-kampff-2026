# Dargk — ELOQUENT 2026 Voight-Kampff Submission

Submission to the **Voight-Kampff** task of the [ELOQUENT Lab 2026](https://eloquent-lab.github.io/) at CLEF 2026.

The Voight-Kampff task asks: *Can text generated by a language model be distinguished from text written by a human?* Participants are given content and style descriptions and must generate ~500-word texts that fool an AI-detection classifier into believing they were human-authored. Systems are scored by how often their outputs pass as human.

Our approach fine-tunes a small LLM with reinforcement learning (GRPO) using an AI-detection classifier as the reward signal, then applies a best-of-N generation strategy at inference time to exploit the stochastic nature of LLMs.

---

## Approach

### Base model

We use [`meta-llama/Llama-3.2-3B-instruct`](https://huggingface.co/meta-llama/Llama-3.2-3B-instruct) — a 3-billion-parameter instruction-tuned decoder-only transformer from Meta's LLaMA 3.2 family.

### Classifier — Mdok2

**Mdok2** is our own AI-detection classifier, inspired by (but distinct from) the original Mdok system. It is a LoRA fine-tuned version of [`FacebookAI/roberta-large`](https://huggingface.co/FacebookAI/roberta-large) (355M parameters, encoder-only) adapted for binary sequence classification (human-written vs. AI-generated). It was trained on the [PAN25 AI-generated text detection dataset (Task 1)](https://pan.webis.de/). Before classification, text is preprocessed by lowercasing and replacing emails, @-mentions, and phone numbers with placeholder tokens.

LoRA configuration for Mdok2: rank r=64, alpha=16, dropout=0.1, targeting query and value projections; classification head trained fully.

### Fine-tuning — GRPO 

We fine-tune the base model using **GRPO** (Group Relative Policy Optimization) via the [TRL](https://github.com/huggingface/trl) library.

The reward function is `1 − p(AI)`, where `p(AI)` is the probability assigned by Mdok2 that a generated text is AI-authored. This is not RLHF — there is no human feedback; the signal comes entirely from the classifier trained on the PAN25 dataset.

Training prompts are drawn from the Voight-Kampff task datasets for **2024, 2025, and 2026**. The model is trained for 10 epochs with GRPO group size G=8.

```bash
python -u llm_finetune_pipelineGRPO_full_mdok_chat_beta.py \
    --epochs 10 \
    --learning_rate 5e-5 \
    --checkpoint_dir ./checkpointsGRPO_full_mdok_beta005_chat \
    --output_dir ./modelGRPO_full_mdok_beta005_chat \
    --datasets 2024,2025,2026 \
    --beta 0.05
```


Pre-trained weights are currently available at [huggingface](https://huggingface.co/jmrodri/Llama-3.2_voight-kampff_beta_005)

### Inference — best-of-N generation

At inference time, for each prompt up to **10 candidate texts** are generated by sampling (temperature=0.8, top_p=0.9), each with a different random seed. After each generation, Mdok2 scores the candidate:

- If `p(AI) < 0.2` (classified as human-written), generation stops early and that text is accepted.
- If no candidate clears the threshold after 10 tries, the one with the lowest `p(AI)` score is selected.

This best-of-N strategy turns the stochasticity of the LLM into a guided search: different seeds produce meaningfully different outputs, and the classifier steers selection toward the most human-like one.

```bash
python Generate_chat.py \
    --finetuned_model modelGRPO_full_mdok_beta005_chat/meta-llama_Llama-3.2-3B-instruct \
    --output grpo_beta005_chat_lora_mdok_epoch10.csv
```

For using the finetuned model available in huggingface, the `--finetuned_model` parameter is `jmrodri/Llama-3.2_voight-kampff_beta_005`.

### Prompt format

Two system prompts are prepended using the model's chat template:

```python
[
    {"role": "system", "content": "You are a helpful assistant that generates helpful answers. "
                                  "You will avoid pleasantries and small talk, focusing on the task at hand."},
    {"role": "system", "content": "You will avoid short paragraphs and bullet points."},
    {"role": "user", "content": prompt},
    {"role": "assistant", "content": ""},
]
```

The user prompt is assembled from the task's base prompt, the `Content` field, and the `Genre and Style` field of each topic.

## Environment — Singularity container

The compute environment is packaged as a [Singularity](https://sylabs.io/singularity/) container defined in `llm.def`. It is based on Python 3.14 and includes PyTorch (CUDA 12.8), Transformers, PEFT, TRL, and all other dependencies.

To build the container (requires `--fakeroot` or root):

```bash
# Optionally submit as a SLURM job
sbatch build.sh

# Or build directly
singularity build --fakeroot llm.sif llm.def
```

---

## Creating a submission

After generating outputs with `Generate_chat.py`, package them into the required PAN zip format:

```bash
python create_submission.py \
    --input grpo_beta005_chat_full_mdok_epoch10.csv \
    --team Dargk \
    --output Dargk.zip
```

Each topic's generated text is trimmed to the last full sentence and written to `Dargk/<id>.txt` inside the zip.
